01-Was ist ein Active Directory

Was ist Active Directory?

Active Directory (AD) ist ein Verzeichnisdienst von Microsoft, mit dem alle Ressourcen hierarchisch gespeichert, identifiziert und zugänglich gemacht werden. Der Aufbau der dahinterliegenden Datenbank orientiert sich am sogenannten x.500-Standard. Zur Abfrage und Modifikation der Datenbankinhalte wird das Lightweight Directory Access Protocol (LDAP) genutzt.

📓 Man kann es sich wie ein Telefonbuch für das Netzwerk vorstellen, in dem alle wichtigen Informationen gespeichert sind.

Er wird hauptsächlich in Unternehmensnetzwerken eingesetzt, um Benutzer, Computer, Gruppen und weitere Ressourcen zentral zu verwalten.

Wichtige Funktionen von Active Directory

  • Zentrale Verwaltung → Benutzerkonten, Passwörter, Computer, Drucker und Gruppenrichtlinien an einem Ort verwalten
  • Authentifizierung → Benutzer melden sich mit einem einzigen Konto im gesamten Netzwerk an („Single Sign-On, SSO“)
  • Autorisierung → Zugriffsrechte für Dateien, Ordner und Anwendungen zentral steuern
  • Gruppenrichtlinien (GPOs) → Regeln und Sicherheitseinstellungen auf Benutzergruppen anwenden
  • Skalierbarkeit → Geeignet für kleine Firmen wie auch für internationale Konzerne (Replikation)

Active Directory Objekte

Die in AD gespeicherten Informationen nennt man AD-Objekte. Es gibt drei Haupttypen:

  1. Benutzer (z. B. Schüler, Lehrer, Mitarbeiter)
  2. Ressourcen (z. B. Computer, Drucker, Server)
  3. Gruppen (Sammlungen von Benutzern, um Rechte einfacher zuzuweisen)

Beispiele für AD-Objekte:
👩 Benutzer → „Max Muster“
🖨️ Ressource → „Drucker 2. Stock“
💼 Gruppe → „Lehrer“

Grundlegende Begriffe

  • Domäne Ein logischer Zusammenschluss von Computern, Benutzern und Ressourcen. Beispiel: schule.local
  • Domänencontroller (DC) Der Server, auf dem Active Directory läuft. Er speichert und verwaltet die Informationen und authentifiziert Benutzer.
  • OU (Organisationseinheit) Eine Art Ordner innerhalb der Domäne. Dient zur logischen Strukturierung, z. B. nach Abteilungen: Lehrer, Schüler, IT.
  • Gruppenrichtlinien (GPOs) Regeln und Einstellungen, die auf Benutzer oder Computer angewendet werden. Beispiel: Automatisches Setzen von Desktop-Hintergrund, Passwortlänge, Softwareinstallation.

Aufbau von Active Directory (Hierarchie)

Gesamte Gesamtstruktur (Forest)
│
└── Domäne (z. B. schule.local)
    │
    ├── Organisationseinheit (OU) Lehrer
    ├── Organisationseinheit (OU) Schüler
    └── Organisationseinheit (OU) IT

Vorteile für Schulen und Unternehmen

✅ Zentrale Benutzerverwaltung (z. B. ein Konto für Schul-PCs, WLAN, Drucker)
✅ Sicherheit durch zentrale Passwortrichtlinien
✅ Zeitersparnis für IT-Administratoren
✅ Bessere Übersicht über Benutzer und Ressourcen

Praxisbeispiel: Schule

  • Jeder Schüler erhält ein persönliches Benutzerkonto (vorname.nachname@schule.local)
  • Beim Einloggen an einem Schul-PC werden automatisch:
    • das richtige Benutzerprofil geladen
    • Netzlaufwerke eingebunden (z. B. H: für persönliche Dateien)
    • Gruppenrichtlinien angewendet (z. B. kein Installieren von Programmen)

Zusammenfassung

Active Directory ist das Herzstück vieler IT-Infrastrukturen. Es ermöglicht die zentrale Verwaltung von Benutzern, Computern und Ressourcen und sorgt für Sicherheit und Effizienz – egal ob in einer Schule oder in einem Grossunternehmen.

Video Zusammenfassung

Active Directory

  • Dienst von Microsoft
  • wird in Windows Server-Umgebungen verwendet
  • hilft bei der Verwaltung von Netzwerken
  • digitales Telefonbuch
  • Informationen über Benutzer, Computer und andere Ressourcen

Wie funktioniert Active Directory

  • Informationen speichern und bereitstellen
  • z.B. auf Drucker zugreifen
  • hierarchische Struktur = "Domäne"
    • weitere Unterkategorien für bessere Organisation

Die Grundlagen der Active Directory Struktur

  • mehrere Schichten
  • ganz oben "Wald"-Ebene
    • eine oder mehrere Domänen
    • weitere Unterkategorien
    • Benutzer, Computer, Drucker
  • eindeutige Namen und Attribute
    • leicht zu finden

Active Directory und Sicherheit

  • Regeln und Richtlinien durch Administratoren
  • Steuerung des Zugangs
  • z.B. bestimmte Benutzer für bestimmte Computer
  • Verschlüsselung
  • weitere Sicherheitsmassnahmen zum Datenschutz

Was ist LDAP

  • Lightweight Directory Access Protocol
  • Standardprotokoll zum Suchen und abrufen von Informationen
  • wird in Active Directory und anderen Verzeichnisdiensten verwendet

Active Directory in der Praxis

  • Einsatz von IT-Abteilungen weltweit
  • Erleichterung der Verwaltung von Netzwerken
  • benutzerfreundliches Zugreifen
  • Hilfe bei Sicherheitsrichtlinien

Thema Einführung

Was ist ein DS?

  • Verzeichnisdienst
  • Informationen zentral verfügbar
  • Strukturiert und organisiert
  • Spart Zeit und erhöht Sicherheit

Typische Eigenschaften?

  • Hierarchische Struktur (ähnlich wie ein Baum)
    • Eine oder mehrere Domänen, weitere Unterkategorien
    • Organisiert Objekte wie Benutzerkonten, Gruppen, Drucker
  • Zentrale Authentifizierung
    • Single Sign-On (SSO)
  • Standardisierte Protokolle
    • z. B. LDAP (Lightweight Directory Access Protocol)

V1: Zentrale Benutzerverwaltung

  • Benutzerkonto einmal anlegen, überall nutzbar
  • Passwortänderungen sofort gültig für alle Systeme

V2: Authentifizierung & Autorisierung

  • Nur berechtigte Benutzer erhalten Zugriff auf Ressourcen
  • Beispiel: Active Directory prüft Login ins Firmennetzwerk

V3: Ressourcenverwaltung

  • Netzwerkdrucker, Server, Ordner zentral verwalten
  • Gruppenrichtlinien zentral steuern

V4: Skalierbarkeit

  • Besonders in grossen Organisationen nützlich
  • Tausende Benutzer und Geräte können zentral organisiert werden
